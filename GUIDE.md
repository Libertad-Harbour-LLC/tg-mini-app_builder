# Как собрать Telegram Mini App с ИИ-ассистентом. Гайд для новичков

Привет! Это пошаговый разбор, как с нуля собрать **мини-приложение внутри
Telegram** (на примере автошколы «Тесла») и встроить в него **ИИ-ассистента**,
который отвечает клиентам на вопросы — и в самом приложении, и в чате с ботом.

Тебе **не нужно уметь программировать**. Весь код напишет ИИ (Claude Code), а ты
будешь просто давать ему понятные команды (мы их все приготовили — копируй и
вставляй) и нажимать кнопки в Telegram и на сайте Vercel.

> 🎯 **Что получится в конце:** у твоего Telegram-бота появится кнопка, которая
> открывает красивое приложение с вкладками (Курсы, Расписание, Ассистент,
> Запись, Контакты). А ИИ будет консультировать людей по твоим ценам, услугам и
> адресам — круглосуточно и бесплатно.

**Цена вопроса:** 0 ₽ (всё на бесплатных тарифах).
**Время:** ~40–60 минут, если впервые.

---

## Что это вообще такое (разбираем слова на пальцах)

Прежде чем начать — пара слов простым языком. Если что-то непонятно сейчас —
ничего, по ходу гайда всё уляжется.

**Telegram Mini App (мини-апп)** — это сайт-приложение, который открывается
**прямо внутри Telegram**, без установки из магазина приложений. Представь
вкладку в браузере, но она живёт в Telegram. Игры в Telegram, магазины,
кошельки — всё это мини-аппы.

**Claude Code** — это ИИ-программист. Ты пишешь ему словами, что хочешь, а он
сам создаёт файлы и пишет код. Работает прямо в браузере на
[claude.com/code](https://claude.com/code). Думай о нём как о подрядчике,
который мгновенно делает то, что ты попросил.

**Скилл (skill)** — это «методичка» для Claude Code. Обычный ИИ может делать
тысячу вещей и иногда «плавает». А когда ты даёшь ему скилл по конкретной теме
(у нас — «как делать Telegram Mini App»), он работает как **узкий специалист**:
знает правильные приёмы и не лепит отсебятину. Файл скилла лежит в этом
репозитории — `telegram-mini-app_skill.md`.

**GitHub** — это «облачная папка» для кода (как Google Диск, но для
программистов). Туда складывается твой проект.

**Деплой (deploy) и хостинг** — звучит страшно, на деле просто. Твой код сам по
себе — это просто файлы в папке. Чтобы приложение **открывалось у людей в
интернете 24/7**, эти файлы должны лежать на каком-то компьютере, который всегда
включён и подключён к сети. Такой компьютер называется **сервер (хостинг)**, а
процесс «выложить код на сервер, чтобы он заработал в интернете» — это и есть
**деплой**. Мы используем бесплатный сервер **Vercel** — он сам берёт код с
GitHub и публикует его в интернете.

**Groq** — это бесплатный «мозг» для нашего ассистента. Это сервис, который даёт
доступ к ИИ-моделям (они и сочиняют ответы). Денег не просит, нужен только
бесплатный ключ.

**Gemini** — ИИ от Google. Мы используем его как помощника, чтобы собрать
«базу знаний» о бизнесе и написать инструкцию для ассистента.

Готов? Поехали по шагам.

## Часть 1. Создаём бота в Telegram и берём токен

Прежде чем делать приложение, нужен сам бот. Боты в Telegram создаются через
специального бота — **@BotFather** (это «паспортный стол» для ботов: через него
бота регистрируют и выдают ему ключ).

### Шаг 1.1. Создаём бота

Открой [@BotFather](https://t.me/BotFather) и отправь по очереди:

```
/start
```
```
/newbot
```

Дальше BotFather спросит:
1. **Имя бота** (Name) — то, что видят люди. Например: `Автошкола Тесла`.
2. **Username** — адрес бота, латиницей, **обязательно заканчивается на `bot`**.
   Например: `tesla_school_bot`.

### Шаг 1.2. Сохраняем токен

После создания BotFather пришлёт **токен** — он выглядит так:

```
8123456789:AAH-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**Токен — это ключ от твоего бота.** Кто им владеет — управляет ботом. Поэтому
**никому его не показывай и не выкладывай в интернет.** Скопируй и сохрани в
заметки — он понадобится позже. Будем называть его **BOT_TOKEN**.

---

## Часть 2. Берём бесплатный ключ Groq (для ИИ)

1. Зайди на https://console.groq.com → войди через Google.
2. В меню выбери **API Keys → Create API Key** → дай имя (например `tesla-bot`)
   → **Submit**.
3. Скопируй ключ (выглядит как `gsk_...`) **сразу** — потом он не покажется
   второй раз. Сохрани в заметки как **GROQ_API_KEY**.

> ✅ **Важно про деньги:** на балансе Groq **ничего не нужно**. Модель
> `llama-3.3-70b-versatile`, которую мы используем, бесплатна (есть лимит по
> скорости запросов, но для бизнеса этого с головой хватает). Если позже
> увидишь, что ассистент молчит — дело почти наверняка **не** в деньгах, а в
> технической мелочи (разберём в разделе «Если что-то не так»).

## Часть 3. Собираем базу знаний и промпт через Gemini (15 минут)

Чтобы ИИ грамотно консультировал, ему нужны две вещи:

1. **База знаний** — все факты о бизнесе: услуги, цены, адреса, ответы на частые
   вопросы. Это «учебник», по которому ассистент отвечает.
2. **Системный промпт** — инструкция, **КАК** себя вести: отвечать на русском,
   ничего не впаривать, а желающих купить — отправлять в раздел «Запись»/
   контакты.

Мы соберём и то, и другое через Gemini. Делается в три простых захода.

### Шаг 3.1. Открываем помощника по составлению промптов

Перейди по ссылке на готового ассистента (Gem), который умеет писать промпты:

```
https://gemini.google.com/gem/1b0FlcxEkGezOA7DVFcf33sX5nKvDa7NC?usp=sharing
```

И отправь ему такое сообщение (можешь скопировать как есть):

```
Мне необходимо написать промт, который зайдёт по ссылке на сайт, детально
проанализирует весь сайт — название, тарифы, услуги, где находится этот
бизнес, что он предоставляет, что делает, адрес, что можно купить — в общем,
всю информацию соберёт именно с этого сайта. Из этой информации необходимо
сформировать базу данных о компании либо об услуге для моего бота, который
будет консультировать людей. База данных должна сформироваться на русском
языке. Пиши промт.
```

Gemini выдаст готовый **промпт для сбора базы знаний**. Скопируй его.

### Шаг 3.2. Собираем базу знаний по своему сайту

Открой **новый чат** в Gemini. Вставь туда промпт из шага 3.1, а следом —
**ссылку на сайт твоего бизнеса**. Gemini проанализирует сайт и выдаст
аккуратный структурированный текст: общая информация, контакты, каталог услуг,
цены, FAQ. **Это и есть твоя база знаний.** Скопируй её целиком в заметки.

> 💡 **Если Gemini не может открыть сайт** (бывает — сайт закрыт от ботов):
> просто скопируй текст со страниц своего сайта руками (цены, услуги, контакты)
> и вставь в чат с просьбой «структурируй это в базу знаний по тому же шаблону».
> Мы в процессе именно так и делали — присылали **скриншоты** страниц.

### Шаг 3.3. Получаем системный промпт для ассистента

Вернись в **первый** чат (где Gemini писал промпт) и отправь такое сообщение, а
в конце вставь свою базу знаний из шага 3.2:

```
Отлично, теперь напиши промпт для моего ИИ-ассистента, который будет
консультировать клиентов по базе знаний. Акцентирую внимание: в промте
ассистент обязан отвечать на русском языке и должен просто отвечать на
вопросы. Ассистент не должен ничего продавать; в случае если человек хочет
что-то приобрести или купить — ИИ-ассистент должен переводить в раздел
контакты, в мини-аппки или в раздел «Оставить заявку».

Вот предварительная база знаний ассистента-консультанта:
<СЮДА ВСТАВЬ СВОЮ БАЗУ ЗНАНИЙ ИЗ ШАГА 3.2>
```

Gemini выдаст готовый **системный промпт** — с ролью, правилами тона, запретом
на продажи и инструкцией, куда направлять желающих купить. **Сохрани его** — мы
отдадим его Claude Code в Части 6.

## Часть 4. Устанавливаем скилл в Claude Code

### Шаг 4.1. Создаём «облачную папку» (репозиторий) на GitHub

Зайди на https://github.com/new → придумай имя (например `tesla-mini-app`) →
выбери **Private** (приватный) → **Create repository**. Готово, это пустая папка
для нашего проекта.

### Шаг 4.2. Открываем Claude Code и подключаем репозиторий

Зайди на [claude.com/code](https://claude.com/code), подключи только что
созданный репозиторий и начни новую сессию (чат).

### Шаг 4.3. Устанавливаем скилл (прикладываем файл)

1. Скачай файл **`telegram-mini-app_skill.md`** (он лежит в этом репозитории, в
   корне).
2. В чате Claude Code **прикрепи этот файл** (значок скрепки / перетащи файл в
   окно).
3. Напиши ровно:

```
установи и активируй скилл
```

Claude Code прочитает файл и положит его в проект (в служебную папку
`.claude/skills/`). Всё — теперь ассистент знает, как правильно делать
мини-аппы.

> ⚠️ **Важный нюанс:** скилл «включается» по-настоящему только в **новой**
> сессии. Поэтому после установки **начни новый чат** в Claude Code — и уже там
> давай команды из Части 5. Это нормально, так и должно быть.

## Часть 5. Просим Claude Code собрать само приложение

Открой **новую** сессию в Claude Code (чтобы скилл был активен) и отправь этот
промпт. Под себя поменяй только описание бизнеса и вкладки:

```
Используя скилл telegram-mini-app, создай Telegram Mini App для моего бизнеса
(автошкола) на React + Vite + TypeScript.

Требования:
- Мобильный-first дизайн под нативную тему Telegram (поддержка светлой и
  тёмной темы автоматически).
- Внизу панель вкладок: Курсы, Расписание, Запись, Контакты.
- Вкладка «Курсы»: категории услуг с тарифами, ценами и списком «что входит».
  У каждого тарифа кнопка «Оставить заявку».
- Вкладка «Запись»: форма (имя, телефон, выбор тарифа, комментарий) с проверкой
  введённых данных. Если отправка заявки боту недоступна — запасной вариант:
  открыть WhatsApp с уже заполненным текстом заявки.
- Вкладка «Контакты»: телефон и кнопки соцсетей (Telegram, WhatsApp, ВКонтакте)
  векторными иконками (SVG), без эмодзи.
- ВЕСЬ контент (цены, тексты, телефоны, ссылки) вынеси в ОДИН файл src/data.ts,
  чтобы я мог легко его редактировать, не трогая код.
- Подготовь проект к деплою на Vercel (добавь vercel.json).

ОЧЕНЬ ВАЖНО (чтобы не было ошибок на Vercel):
В проекте будут серверные функции в папке api, и проект работает в режиме
ES-модулей ("type": "module"). Поэтому ВО ВСЕХ относительных импортах в папке
api ОБЯЗАТЕЛЬНО указывай расширение .js — например:
import { x } from './_lib/telegram.js';
Если не указать .js, на Vercel будет ошибка ERR_MODULE_NOT_FOUND и функции
упадут. Учти это с самого начала.
```

Claude всё соберёт за пару минут. Не вникай в код — главное, чтобы он сказал
«готово».

> 🧠 **Почему мы заранее написали про `.js`?** Это та самая ошибка, на которой мы
> споткнулись в реальном процессе (см. раздел «Если что-то не так», ошибка №1).
> Заложив требование в промпт сразу, ты её **полностью избежишь**.

### Шаг 5.1. Заливаем реальные данные бизнеса

Теперь подставим настоящие цены и контакты. Сделай скриншоты страниц своего
сайта с тарифами и контактами и отправь их в чат с таким сообщением:

```
Вот реальные цены и контакты моего бизнеса (на скриншотах). Перенеси их
аккуратно в src/data.ts: категории, тарифы, что входит, цены, телефон и ссылки
на соцсети.
```

(прикрепи скриншоты)

> 💡 Скриншоты — самый надёжный способ: Claude точно перенесёт данные без
> выдумок. Позже, чтобы что-то поменять (например, поднять цену), просто пиши:
> «поменяй цену тарифа Стандарт на 9500 в src/data.ts».

## Часть 6. Добавляем ИИ-ассистента (Groq)

Теперь — «мозги». Отправь Claude Code этот промпт, а **в конце вставь системный
промпт**, который тебе выдал Gemini в Части 3:

```
Добавь в проект ИИ-ассистента на бесплатной модели Groq. Нужно в двух местах:

1) Новая вкладка «Ассистент» в мини-аппе — это чат, где пользователь пишет
   вопрос и получает ответ от ИИ.
2) Ответы в самом боте: serverless-функция api/bot.ts (это webhook), которая
   получает сообщения пользователей боту и отвечает на них через Groq.

Технические условия:
- Ключ Groq бери ТОЛЬКО из переменной окружения GROQ_API_KEY (на сервере),
  никогда не вписывай ключ в код.
- Модель по умолчанию: llama-3.3-70b-versatile (бесплатная).
- Системный промпт и базу знаний помести в отдельный файл api/_lib/assistant.ts.
- Во всех относительных импортах в папке api используй расширение .js
  (например './_lib/assistant.js') — иначе на Vercel будет ERR_MODULE_NOT_FOUND.

Вот системный промпт для ассистента (вставь его в api/_lib/assistant.ts как есть):

<СЮДА ВСТАВЬ СИСТЕМНЫЙ ПРОМПТ ИЗ ЧАСТИ 3.3>
```

Готово — теперь у приложения есть вкладка с ИИ, а у бота — умные ответы.

### Шаг 6.1. Отправляем всё в основную версию (merge в main)

Маленькое, но важное объяснение. Claude Code работает в «черновике» (это
называется **ветка**), а сервер Vercel публикует только «чистовик» (ветка
**`main`**). Пока изменения не переехали из черновика в чистовик, в приложении
их **не будет**.

Поэтому в конце просто напиши Claude:

```
Создай PR и смержи всё в main.
```

(PR — это «запрос на слияние», а merge — само слияние черновика в чистовик. Claude
сделает это сам.)

## Часть 7. Деплой на Vercel (публикуем приложение в интернет)

### Шаг 7.1. Импортируем проект

1. Зайди на [vercel.com](https://vercel.com) → **Add New… → Project**.
2. Найди свой репозиторий → **Import**.
3. Vercel сам поймёт, что это Vite-приложение. Ничего менять не надо.
4. **Пока не нажимай Deploy** — сначала добавим ключи (следующий шаг).

### Шаг 7.2. Добавляем переменные окружения (ключи)

Маленькая аналогия. Ключи (токен бота, ключ Groq) — это как **ключи от сейфа**.
Их **нельзя** писать прямо в коде (код лежит в открытом GitHub — это всё равно
что приклеить ключ от квартиры на дверь). Вместо этого ключи кладут в защищённое
место на сервере — это и есть **переменные окружения (Environment Variables)**.

В Vercel: твой проект → **Settings → Environment Variables**. Добавь:

| Имя (Key) | Значение (Value) | Откуда взять |
|-----------|------------------|--------------|
| `TELEGRAM_BOT_TOKEN` | токен бота `8123...:AAH...` | Часть 1 |
| `GROQ_API_KEY` | ключ `gsk_...` | Часть 2 |
| `GROQ_MODEL` | `llama-3.3-70b-versatile` | (необязательно) |

Для каждой включи тумблер **Sensitive** (значит «секрет») и оставь окружение
**Production** (можно и Preview). Жми **Save** для каждой.

![Окно Add Environment Variable в Vercel: поля Key, Value, тумблер Sensitive, кнопка Save](docs/images/09-vercel-add-env.png)

![Список переменных проекта: TELEGRAM_BOT_TOKEN и GROQ_API_KEY](docs/images/09-vercel-env-list.png)

### Шаг 7.3. Деплоим

Нажми **Deploy** (или в разделе **Deployments** у верхнего билда **⋯ →
Redeploy**). Подожди статус **Ready** (~1 минута).

> ⚠️ **Запомни правило (грабли №3):** Vercel применяет переменные **только к
> новым сборкам**. Каждый раз, когда добавил или поменял переменную — жми
> **Redeploy**, иначе сервер «не увидит» ключ.

Скопируй адрес своего приложения — он вида:

```
https://имя-проекта.vercel.app
```
(у нас был `https://tesla-mini-app.vercel.app`). Будем звать его **APP_URL**.

![Обзор проекта в Vercel: блок Production Deployment и адрес в разделе Domains](docs/images/08-vercel-overview.png)

## Часть 8. Подключаем приложение к боту через BotFather

Теперь привяжем приложение к боту, чтобы у него появилась кнопка запуска.

Открой [@BotFather](https://t.me/BotFather) → `/mybots` → выбери своего бота.
Нужный раздел называется **Mini Apps** (значок из четырёх квадратиков).

![Меню бота в BotFather: Edit Info, Commands, Mini Apps, Bot Settings и др.](docs/images/10-botfather-bot-menu.png)

⚠️ **Грабли №4:** настройки приложения лежат именно в **Mini Apps**, а не в
«Bot Settings». В «Bot Settings» ты попадёшь на страницу с тумблерами (Inline
Mode и т.п.) — это **не то**. Если заблудился — вернись назад и ищи **Mini Apps**.

![Страница Bot Settings с тумблерами — это НЕ то место для Mini App](docs/images/10-botfather-bot-settings-modes.png)

Внутри **Mini Apps** два пункта:

![Экран Mini Apps: пункты Menu Button и Main App, Direct Links](docs/images/10-botfather-mini-apps.png)

### Шаг 8.1. Main App (приложение бота)

1. **Mini Apps → Main App → Enable** (или «задать URL»).
2. Вставь свой **APP_URL** обязательно **со слешем на конце**:
   ```
   https://tesla-mini-app.vercel.app/
   ```
3. **Launch Mode** выбери **Fullsize** (приложение на всю высоту — нашим
   вкладкам внизу нужно место). *Compact* — половина экрана, *Fullscreen* — во
   весь экран без шапки Telegram.
4. **Save**.

![Экран Main App: поле Enter URL, выбор Launch Mode, кнопка Save](docs/images/10-botfather-main-app.png)

### Шаг 8.2. Menu Button (кнопка слева от поля ввода)

1. **Mini Apps → Menu Button**.
2. Вставь тот же APP_URL со слешем: `https://tesla-mini-app.vercel.app/`
3. Введи текст кнопки, например `Открыть автошколу`.
4. **Save**.

![Экран Menu Button: предпросмотр кнопки, поле URL, текст кнопки, Save](docs/images/10-botfather-menu-button.png)

> ⚠️ URL всегда **корневой и со слешем** (`.../`). Если позже подключишь свой
> домен (Vercel → Settings → Domains) — указывай его вместо `*.vercel.app`.

## Часть 9. Включаем ИИ-ответы в самом боте (webhook)

**Что такое webhook простыми словами.** Чтобы бот отвечал на сообщения в чате,
Telegram должен знать, **куда** пересылать эти сообщения. Webhook — это просто
адрес, на который Telegram будет слать каждое новое сообщение. У нас это наша
функция `/api/bot` на Vercel. Настраивается один раз — открытием специальной
ссылки в браузере.

Открой в браузере такую ссылку, подставив **свой BOT_TOKEN** и **свой APP_URL**:

```
https://api.telegram.org/bot<BOT_TOKEN>/setWebhook?url=<APP_URL>/api/bot
```

Пример с фейковыми значениями:

```
https://api.telegram.org/bot8123456789:AAH.../setWebhook?url=https://tesla-mini-app.vercel.app/api/bot
```

В ответ браузер должен показать:

```json
{"ok":true,"result":true,"description":"Webhook was set"}
```

![Ответ браузера: {"ok":true,"result":true,"description":"Webhook was set"}](docs/images/11-setwebhook-result.png)

Полезные ссылки на будущее (открываются так же в браузере):
- Проверить статус: `https://api.telegram.org/bot<BOT_TOKEN>/getWebhookInfo`
- Отключить: `https://api.telegram.org/bot<BOT_TOKEN>/deleteWebhook`

## Если что-то не так (частые проблемы и решения)

**Где смотреть причину:** Vercel → твой проект → **Logs** (или Deployments →
последний билд → Logs). Там видно ошибки серверных функций. Именно логи спасли
нас в реальном процессе.

| Симптом | В чём причина и что делать |
|---------|----------------------------|
| **Ассистент молчит, в логах `ERR_MODULE_NOT_FOUND`** | Ошибка №1 (см. ниже). В импортах в папке `api` не хватает `.js`. Скажи Claude: «во всех относительных импортах в папке api добавь расширение .js», затем merge в main. |
| **Новой вкладки/функции нет в приложении** | Изменения остались в черновике. Скажи Claude «создай PR и смержи в main» — Vercel пересоберёт. |
| **Ассистент молчит, в логах про ключ/`GROQ`** | Не добавил `GROQ_API_KEY` в Vercel или забыл **Redeploy** после добавления. Добавь и сделай Redeploy. |
| **Бот не отвечает в чате** | Открой `getWebhookInfo` — в поле `last_error_message` будет причина. Часто — деплой ещё не `Ready` или неверный APP_URL в `setWebhook`. |
| **В BotFather не вижу пункт Mini App** | Ты в «Bot Settings». Вернись и зайди в **Mini Apps**. |
| **Приложение открывается, но пустое/битое** | Открой APP_URL прямо в браузере — если и там пусто, смотри Vercel → Logs билда. Скажи Claude текст ошибки. |
| **Заявка из формы «не доходит»** | Это нормально при запуске через Main App. Сработает запасной путь — заявка уходит в WhatsApp с готовым текстом. |

### Ошибка №1 — главная, разбираем подробно

**Что было:** ассистент молчал, а в логах Vercel у функций горело красным:
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module '/var/task/api/_lib/telegram'
```

![Логи Vercel: строки 500 с ERR_MODULE_NOT_FOUND](docs/images/13-vercel-logs-error.png)

**Почему так вышло (на пальцах):** наш проект работает в режиме «ES-модулей».
В этом режиме, когда один файл подключает другой файл рядом, он **обязан**
называть его с расширением `.js`. Без расширения сервер «не находит» файл и
функция падает — ещё **до** того, как обратится к ИИ. Поэтому казалось, что
«виноват ИИ или баланс», хотя проблема была чисто техническая, в одной букве.

**Было (неправильно):**
```ts
import { askGroq } from './_lib/assistant';
```
**Стало (правильно):**
```ts
import { askGroq } from './_lib/assistant.js';
```

**Как избежать:** мы зашили это требование прямо в промпты Части 5 и 6 — если ты
их скопировал как есть, ошибки не будет. Если всё же поймал — просто скажи Claude:
«во всех относительных импортах в папке api добавь расширение .js», и смержи в
main.

## Как переделать гайд под свой бизнес

Архитектура одна — меняется только наполнение:

1. **Контент:** скажи Claude поменять данные в `src/data.ts` (цены, услуги,
   адреса, телефон, соцсети) — или пришли новые скриншоты.
2. **База знаний и промпт:** повтори Часть 3 для своего сайта и попроси Claude
   обновить `api/_lib/assistant.ts` новым промптом и базой.
3. **Свой бот и ключи:** свой `TELEGRAM_BOT_TOKEN` и `GROQ_API_KEY` в Vercel.
4. **Подключение:** Части 7–9 (деплой, BotFather, webhook) — без изменений.

Любую правку можно просить у Claude человеческими словами: «поменяй цены»,
«обнови базу знаний ассистента», «создай PR и смержи в main». Главное — помни про
грабли из раздела «Если что-то не так», и всё соберётся гладко.

## Шпаргалка (весь путь в 12 строк)

1. Завести аккаунты: GitHub, Vercel, Groq, Gemini; создать бота в @BotFather.
2. Сохранить **BOT_TOKEN** (Часть 1) и **GROQ_API_KEY** (Часть 2).
3. Через Gemini собрать **базу знаний** и **системный промпт** (Часть 3).
4. Создать репозиторий на GitHub.
5. В Claude Code приложить `telegram-mini-app_skill.md` → «установи и активируй скилл».
6. В **новой** сессии — промпт из Части 5 (создать приложение, с требованием `.js`).
7. Прислать скриншоты с ценами → залить в `src/data.ts`.
8. Промпт из Части 6 — добавить ИИ-ассистента + вставить системный промпт.
9. «Создай PR и смержи в main».
10. Vercel: импорт репозитория → переменные (`TELEGRAM_BOT_TOKEN`, `GROQ_API_KEY`) → Deploy.
11. BotFather → **Mini Apps** → Main App + Menu Button (APP_URL со слешем).
12. Открыть ссылку `setWebhook` для `/api/bot`. Проверить бота. Готово!
