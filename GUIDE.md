# Как создать Telegram Mini App с ИИ-ассистентом — гайд для новичков

Полный пошаговый разбор на примере мини-аппа для **автошколы «Тесла»**: от
установки скилла до работающего бота с ИИ-консультантом на бесплатной модели
Groq. Гайд рассчитан на человека, который **не программист** и **не разбирается
в ИИ**. Все «грабли», на которые мы наступили, вынесены в отдельные
предупреждения ⚠️, чтобы вы их обошли.

> Что получится в итоге: приложение внутри Telegram с вкладками
> «Курсы / Расписание / Ассистент / Запись / Контакты», где ИИ отвечает на
> вопросы клиентов по базе знаний компании, плюс тот же ИИ отвечает в самом
> чате с ботом.

<a name="1-что-такое-скилл"></a>
## 1. Что такое скилл и зачем он нужен

Мы делаем приложение не вручную, а с помощью **Claude Code** — это ИИ-ассистент
для написания кода. Им можно пользоваться прямо в браузере на
[claude.com/code](https://claude.com/code) («Claude Code on the web»): он сам
создаёт файлы, пишет код, делает коммиты в ваш репозиторий.

**Скилл (skill)** — это «методичка» для Claude Code: набор знаний и шаблонов по
конкретной теме. Когда скилл установлен, ассистент знает правильные паттерны и
не изобретает велосипед.

Мы использовали скилл **`telegram-mini-app`** — он описывает, как устроены
Telegram Mini Apps: подключение Telegram Web App API, нативная тема (светлая/
тёмная), мобильный-first дизайн, формы записи, проверка пользователя и т.д.

Источник скилла (файл `SKILL.md`):
```
https://github.com/tapanshah/antigravity-aswsome-skills/blob/38a0b4fdd92311354b04ce3890c13cf3149b1576/skills/telegram-mini-app/SKILL.md
```

> 💡 Скилл — это просто текстовый файл с инструкциями. Его не «запускают» как
> программу: он становится доступен ассистенту и применяется автоматически,
> когда задача подходит по теме.

<a name="2-подготовка"></a>
## 2. Что понадобится завести заранее (аккаунты и программы)

Всё бесплатно. Заведите по порядку:

| Что | Зачем | Ссылка |
|-----|-------|--------|
| Аккаунт **GitHub** | Хранилище кода (репозиторий) | https://github.com |
| Аккаунт **Vercel** | Бесплатный хостинг приложения и серверных функций | https://vercel.com |
| **Telegram** + **@BotFather** | Создать бота и подключить мини-апп | https://t.me/BotFather |
| Аккаунт **Groq** | Бесплатный ИИ (модель отвечает на вопросы) | https://console.groq.com |
| **Google Gemini** | Поможет собрать базу знаний и написать промпт | https://gemini.google.com |
| **Claude Code** | Сам пишет код приложения | https://claude.com/code |

Программы на компьютер ставить **не обязательно**, если работаете через
Claude Code в браузере — код собирается в облаке. Если хотите запускать локально,
понадобится **Node.js 20+** (https://nodejs.org) и **git**.

### 2.1. Создаём бота в Telegram
1. Откройте [@BotFather](https://t.me/BotFather) → команда `/newbot`.
2. Задайте имя и username (должен заканчиваться на `bot`, напр. `tesla_school_bot`).
3. BotFather пришлёт **токен** вида `123456789:AAE...` — сохраните, он секретный.

### 2.2. Получаем ключ Groq
1. Зайдите на https://console.groq.com → раздел **API Keys** → **Create API Key**.
2. Скопируйте ключ вида `gsk_...`.

> ✅ **Важно про Groq:** деньги на балансе НЕ нужны. Модели (например
> `llama-3.3-70b-versatile`) доступны бесплатно с лимитами по скорости. Ключа
> достаточно.

<a name="3-установка-скилла"></a>
## 3. Установка скилла telegram-mini-app

1. Создайте на GitHub пустой репозиторий (напр. `tg-mini-app_builder`).
2. Откройте [Claude Code на вебе](https://claude.com/code), подключите этот
   репозиторий и создайте сессию.
3. Дайте ассистенту задачу — буквально вставьте ссылку на скилл:

   > «Установи этот скилл: `https://github.com/tapanshah/antigravity-aswsome-skills/blob/38a0b4fdd92311354b04ce3890c13cf3149b1576/skills/telegram-mini-app/SKILL.md`»

Ассистент скачает содержимое и положит его в проект по пути:
```
.claude/skills/telegram-mini-app/SKILL.md
```
После коммита скилл «прописан» в проекте. В новых сессиях он подхватывается
автоматически.

> ⚠️ **Нюанс:** скилл, установленный в середине сессии, станет «активным» только
> в **новой** сессии Claude Code. Это нормально.

<a name="4-создаём-приложение"></a>
## 4. Создаём само приложение (мини-апп)

Дайте ассистенту задачу: создать Telegram Mini App под ваш бизнес. Мы выбрали
стек **React + Vite + TypeScript** — современный и удобный для расширения.

Что ассистент создаёт (структура проекта):
```
.
├── api/                      # серверные функции (работают на Vercel)
│   ├── _lib/                 # общий код для функций
│   │   ├── assistant.ts      # системный промпт + база знаний + вызов Groq
│   │   └── telegram.ts       # проверка Telegram, отправка сообщений
│   ├── verify.ts             # проверка подписи пользователя (initData)
│   ├── chat.ts               # ассистент для мини-аппа
│   └── bot.ts                # ИИ-ответы в самом боте (webhook)
├── src/
│   ├── components/           # экраны: Courses, Schedule, EnrollForm, Contacts, Assistant, icons
│   ├── App.tsx               # навигация по вкладкам
│   ├── data.ts               # ВСЕ тексты/цены/контакты в одном месте
│   ├── useTelegram.ts        # подключение Telegram (тема, кнопки, вибрация)
│   ├── api.ts                # запросы с фронта к /api/*
│   └── index.css             # стили (мобильный-first, тема Telegram)
├── index.html
├── package.json
├── vite.config.ts
└── vercel.json               # настройки деплоя на Vercel
```

Ключевые принципы, которые задаёт скилл:
- **mobile-first** (95% Telegram — телефоны);
- **нативная тема** Telegram (светлая/тёмная) через `tg.themeParams`;
- индикаторы загрузки, нижняя панель вкладок, вибро-отклик (haptics).

> 💡 Весь контент специально вынесен в один файл `src/data.ts` — чтобы менять
> цены и тексты, не трогая код.

<a name="5-данные-бизнеса"></a>
## 5. Наполняем приложение реальными данными бизнеса

Сначала ассистент собрал «рыбу» с примерными ценами. Потом мы заменили на
реальные.

⚠️ **Грабли №1 — сайт не открывался автоматически.** Облачное окружение
Claude Code имеет сетевые ограничения: наш сайт `автошколатесла.рф` отдавал
`403` / «Host not in allowlist». То есть ассистент **не смог** прочитать сайт сам.

**Решение:** мы просто прислали ассистенту **скриншоты** страниц с ценами и
текстом контактов. Он перенёс данные вручную в `src/data.ts`. Это надёжный путь:
сфотографируйте/заскриньте свои тарифы и пришлите ассистенту.

Так в `data.ts` появились категории B / A / C с тарифами, доп. услугами,
телефон `+7 (939) 505 02-10` и ссылки на соцсети (Telegram, WhatsApp, ВК, MAX).

> 💡 Иконки соцсетей сделали векторными (SVG) в `src/components/icons.tsx` —
> выглядят чётко на любом экране, в отличие от эмодзи.

<a name="6-база-знаний"></a>
## 6. Готовим базу знаний и системный промпт через Gemini

Чтобы ИИ грамотно консультировал, ему нужны две вещи:
1. **База знаний** — факты о компании (услуги, цены, адреса, FAQ).
2. **Системный промпт** — инструкция, КАК отвечать (на русском, без «впаривания»,
   при желании купить — отправлять в раздел «Оставить заявку»/контакты).

Мы собрали и то, и другое через Google Gemini. Точные шаги (можете повторить
дословно):

### Шаг 6.1. Промпт-генератор
Открыли специального помощника по составлению промптов (Gem):
```
https://gemini.google.com/gem/1b0FlcxEkGezOA7DVFcf33sX5nKvDa7NC?usp=sharing
```
И написали ему:

> «Мне необходимо написать промт, который зайдёт по ссылке на сайт, детально
> проанализирует весь сайт — название, тарифы, услуги, где находится этот
> бизнес, что он предоставляет, что делает, адрес, что можно купить — в общем,
> всю информацию соберёт именно с этого сайта. Из этой информации необходимо
> сформировать базу данных о компании либо об услуге для моего бота, который
> будет консультировать людей. База данных должна сформироваться на русском
> языке. Пиши промт».

Gemini выдал готовый промпт для сбора базы знаний.

### Шаг 6.2. Собираем базу знаний
Открыли **новый чат** в Gemini, вставили туда полученный промпт, **приложили
ссылку на сайт** автошколы — и Gemini сформировал структурированный текст:
общая информация, контакты, каталог услуг, тарифы, процесс покупки, FAQ. Это и
есть наша база знаний (тот самый большой текст «БАЗА ДАННЫХ КОМПАНИИ ДЛЯ
ЧАТ-БОТА»).

> 💡 Если Gemini не может зайти на сайт — скопируйте текст со страниц вручную и
> вставьте в чат, попросив структурировать по тому же шаблону.

### Шаг 6.3. Получаем системный промпт ассистента
Вернулись в **первый** чат (где генерировали промпт) и написали:

> «Отлично, теперь напиши промпт для моего ИИ-ассистента, который будет
> консультировать клиентов по базе знаний. Акцентирую внимание: в промте
> ассистент обязан отвечать на русском языке и должен просто отвечать на
> вопросы. Ассистент не должен ничего продавать; в случае если человек хочет
> что-то приобрести или купить — ИИ-ассистент должен переводить в раздел
> контакты, в мини-аппки или в раздел "Оставить заявку".
>
> Вот предварительная база знаний ассистента-консультанта: <вставили текст из
> шага 6.2>».

Gemini выдал **системный промпт** с ролью, правилами тона, запретом на продажи и
«протоколом маршрутизации» (куда направлять желающих купить). Этот промпт вместе
с базой знаний мы и поместили в файл `api/_lib/assistant.ts`.

> 💡 Итог: `assistant.ts` содержит `SYSTEM_PROMPT` (роль + правила) и внутри него
> блок `<knowledge_base>` с фактами. Чтобы обновить информацию о компании —
> редактируете этот файл.

<a name="7-ии-ассистент"></a>
## 7. Добавляем ИИ-ассистента (Groq)

ИИ работает в двух местах, и оба обращаются к Groq через **серверные функции**
(ключ Groq хранится только на сервере и никогда не попадает в браузер).

**Как устроено:**
- `api/_lib/assistant.ts` — системный промпт, база знаний и функция `askGroq()`,
  которая шлёт запрос в Groq и возвращает ответ. Модель по умолчанию —
  `llama-3.3-70b-versatile` (бесплатная).
- `api/chat.ts` — принимает сообщения из мини-аппа, добавляет системный промпт,
  зовёт `askGroq`, возвращает ответ. В мини-аппе это вкладка **«Ассистент»**
  (`src/components/Assistant.tsx`).
- `api/bot.ts` — «вебхук»: Telegram присылает сюда каждое сообщение боту, функция
  отвечает текстом от ИИ.

Логика вызова Groq (упрощённо, для понимания — пишет ассистент сам):
```ts
const res = await fetch('https://api.groq.com/openai/v1/chat/completions', {
  method: 'POST',
  headers: {
    'Content-Type': 'application/json',
    Authorization: `Bearer ${process.env.GROQ_API_KEY}`,
  },
  body: JSON.stringify({
    model: process.env.GROQ_MODEL || 'llama-3.3-70b-versatile',
    temperature: 0.3,
    messages: [{ role: 'system', content: SYSTEM_PROMPT }, ...history],
  }),
});
```

> ✅ Бесплатная модель указывается в переменной `GROQ_MODEL` (необязательно — по
> умолчанию уже стоит бесплатная `llama-3.3-70b-versatile`). Баланс не нужен.

<a name="8-деплой"></a>
## 8. Заливаем проект на GitHub и деплоим на Vercel

### 8.1. Git: ветка → коммит → PR → merge
Claude Code работает в отдельной **ветке** (напр. `claude/clever-...`) и делает
коммиты. Чтобы изменения попали в основную ветку `main`, создаётся
**Pull Request (PR)** и выполняется **merge**.

⚠️ **Грабли №2 — деплой идёт из `main`, а код в ветке.** Vercel собирает прод
из ветки `main`. Если фича закоммичена в рабочую ветку, но **не смержена** в
`main`, в приложении её НЕ будет (у нас так «пропадала» вкладка «Ассистент»).

**Правило:** после готовности фичи — обязательно **merge ветки в `main`**
(через PR). Тогда Vercel автоматически пересоберёт прод. Просто попросите
Claude Code: «создай PR и смержи в main».

### 8.2. Импорт проекта в Vercel
1. На [vercel.com](https://vercel.com) → **Add New… → Project**.
2. Выберите ваш GitHub-репозиторий → **Import**.
3. Vercel сам определит фреймворк **Vite**. Настройки по умолчанию подходят:
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`
4. Перед первым деплоем добавьте переменные окружения (см. раздел 9) → **Deploy**.

После деплоя Vercel даст адрес вида:
```
https://<имя-проекта>.vercel.app
```
(у нас: `https://tg-mini-app-builder.vercel.app`).

Файл `vercel.json` уже настроен: фреймворк Vite + правило, чтобы все «маршруты»
кроме `/api/...` отдавали приложение (SPA).

<a name="9-переменные"></a>
## 9. Переменные окружения в Vercel

Vercel → ваш проект → **Settings → Environment Variables**. Добавьте для
окружения **Production** (и при желании Preview):

| Имя | Обяз. | Значение | Зачем |
|-----|-------|----------|-------|
| `TELEGRAM_BOT_TOKEN` | да | токен из @BotFather (`123...:AAE...`) | проверка пользователя + ответы бота |
| `GROQ_API_KEY` | да (для ИИ) | ключ из console.groq.com (`gsk_...`) | вызов ИИ-модели |
| `GROQ_MODEL` | нет | напр. `llama-3.3-70b-versatile` | сменить модель (по умолчанию бесплатная) |
| `TELEGRAM_WEBHOOK_SECRET` | нет | любая случайная строка | защита вебхука бота |

Все значения — **секреты**, держите их только в Vercel (включайте тумблер
**Sensitive**). В код их вписывать нельзя.

⚠️ **Грабли №3 — после добавления переменных нужен Redeploy.** Vercel применяет
новые переменные только к **новым** сборкам. Добавили/поменяли переменную →
**Deployments → у верхнего деплоя меню ⋯ → Redeploy** (или сделайте merge в
`main`, что тоже запустит сборку). Без этого функции «не увидят» переменную.

> Если не хотите `TELEGRAM_WEBHOOK_SECRET` — просто не создавайте эту переменную
> и не передавайте `secret_token` в шаге 11. Проверка секрета включается, только
> если переменная задана.

<a name="10-botfather"></a>
## 10. Подключаем приложение к боту через BotFather

Откройте [@BotFather](https://t.me/BotFather) → `/mybots` → выберите бота. Нужный
раздел — **Mini Apps** (иконка из четырёх квадратиков). Там два пункта:

### 10.1. Main App (приложение бота)
1. **Mini Apps → Main App → Enable / задать URL**.
2. Вставьте **корневой** адрес деплоя со слешем на конце:
   ```
   https://tg-mini-app-builder.vercel.app/
   ```
3. **Launch Mode** выберите **Fullsize** (приложение на всю высоту — у нас внизу
   панель вкладок, ей нужно место). *Compact* — половина экрана, *Fullscreen* —
   во весь экран без шапки.
4. **Save**.

### 10.2. Menu Button (кнопка слева от поля ввода)
1. **Mini Apps → Menu Button**.
2. Тот же URL: `https://tg-mini-app-builder.vercel.app/`
3. Текст кнопки, напр. `Открыть автошколу`.
4. **Save**.

⚠️ **Грабли №4 — где искать пункты в BotFather.** Интерфейс обновлённый. Пункты
«Main App» и «Menu Button» лежат именно в разделе **Mini Apps**, а не в
«Bot Settings» (там вы попадёте на страницу тумблеров Inline/Group и т.п.). Если
заблудились — вернитесь к списку и ищите **Mini Apps**.

⚠️ **Важно:** URL всегда **корневой, со слешем** (`.../`). Если подключите
кастомный домен в Vercel (Settings → Domains), используйте его вместо
`*.vercel.app`.

<a name="11-webhook"></a>
## 11. Включаем ИИ-ответы в самом боте (webhook)

Чтобы бот отвечал на сообщения в чате (а не только в мини-аппе), нужно один раз
сказать Telegram, куда слать сообщения — на нашу функцию `/api/bot`. Это делается
открытием специальной ссылки в браузере (подставьте свой токен):

**Без секрета:**
```
https://api.telegram.org/bot<ТОКЕН>/setWebhook?url=https://tg-mini-app-builder.vercel.app/api/bot
```

**С секретом** (если задали `TELEGRAM_WEBHOOK_SECRET`):
```
https://api.telegram.org/bot<ТОКЕН>/setWebhook?url=https://tg-mini-app-builder.vercel.app/api/bot&secret_token=<ВАШ_СЕКРЕТ>
```

В ответ браузер покажет: `{"ok":true,"result":true,"description":"Webhook was set"}`.

Полезные ссылки для проверки/сброса:
- Проверить: `https://api.telegram.org/bot<ТОКЕН>/getWebhookInfo`
- Снять: `https://api.telegram.org/bot<ТОКЕН>/deleteWebhook`

> 💡 Бот отвечает на любое текстовое сообщение (кроме `/start` и `/help`) и не
> хранит историю между сообщениями — каждый вопрос обрабатывается отдельно.

## 12. Проверка, что всё работает

После того как в Vercel верхний деплой из `main` стал **Ready**:

<a name="13-ошибки"></a>
## 13. ⚠️ Все ошибки, которые мы поймали, и как их избежать

### Ошибка №1 (главная) — функции падали с `ERR_MODULE_NOT_FOUND`
**Симптом:** ассистент молчит; в Vercel → Logs у `/api/chat`, `/api/bot`,
`/api/verify` статус **500** и текст:
```
Error [ERR_MODULE_NOT_FOUND]: Cannot find module '/var/task/api/_lib/telegram'
imported from /var/task/api/verify.js
```
**Причина:** в `package.json` стоит `"type": "module"`, поэтому Vercel собирает
серверные функции как **ES-модули (ESM)**. А Node.js в режиме ESM требует
**явное расширение `.js`** в относительных импортах. Импорты были написаны без
расширения — при сборке всё ок, но **в рантайме** модуль не находится, и функция
падает ещё ДО обращения к Groq. Поэтому казалось, что «виноват ИИ/баланс», хотя
проблема была чисто техническая.

**Неправильно:**
```ts
import { verifyInitData } from './_lib/telegram';
import { askGroq } from './_lib/assistant';
```
**Правильно (добавить `.js`):**
```ts
import { verifyInitData } from './_lib/telegram.js';
import { askGroq } from './_lib/assistant.js';
```
> Расширение `.js` (а не `.ts`) пишется специально: после компиляции файл станет
> `.js`, и путь должен указывать на итоговый файл. Это стандартное требование
> ESM. **Сразу пишите импорты в `api/` с `.js`** — и этой ошибки не будет.

### Ошибка №2 — «фича пропала» (не смержили в `main`)
Прод собирается из ветки `main`. Изменения в рабочей ветке не появятся в
приложении, пока не сделан **merge в `main`**. Всегда мержите готовую фичу.

### Ошибка №3 — переменные не подхватились
После добавления/изменения переменных в Vercel обязательно **Redeploy**
(или merge в `main`). Старый деплой переменные не видит.

### Ошибка №4 — сайт не читается автоматически
Облако Claude Code и Gemini могут не открыть ваш сайт (403/ограничения сети).
Тогда передавайте данные **скриншотами или текстом** вручную.

### Ошибка №5 — заявки из формы «не доходят» боту
Telegram доставляет данные из мини-аппа боту через `sendData` **только** если
приложение открыто **reply-keyboard кнопкой** `web_app`. При открытии через
Main App / Menu Button `sendData` не работает. Поэтому в форме записи сделан
**запасной путь**: заявка уходит менеджеру в **WhatsApp** с заполненным текстом.

### Ошибка №6 — не там искали настройки в BotFather
Main App и Menu Button — в разделе **Mini Apps**, а не в «Bot Settings».

### Не-ошибка — Groq бесплатен
Баланс для Groq не нужен. Если ИИ молчит — причина в логах (чаще всего ошибка
№1 или №3), а не в деньгах.

<a name="14-под-себя"></a>
## 14. Как переделать гайд под свой бизнес

1. **Контент:** отредактируйте `src/data.ts` — название, тарифы, цены, адреса,
   телефон, соцсети.
2. **База знаний и промпт:** повторите раздел 6 для своего сайта и вставьте новый
   `SYSTEM_PROMPT` + базу в `api/_lib/assistant.ts`.
3. **Бот:** создайте своего бота в @BotFather, подставьте его токен в Vercel,
   подключите Main App + Menu Button (раздел 10) и вебхук (раздел 11).
4. **Ключ Groq:** свой `GROQ_API_KEY`.
5. **Деплой:** свой репозиторий на GitHub → импорт в Vercel → переменные →
   Deploy.

Каждую задачу можно поручить Claude Code человеческими словами: «поменяй цены в
data.ts», «обнови базу знаний ассистента», «создай PR и смержи в main». Главное —
помнить про шесть «граблей» выше, и всё соберётся с первого раза.

### Краткая шпаргалка (порядок действий)
1. Завести аккаунты: GitHub, Vercel, Telegram-бот, Groq, Gemini.
2. Установить скилл `telegram-mini-app` в репозиторий.
3. Создать мини-апп (React+Vite+TS) через Claude Code.
4. Заполнить `src/data.ts` реальными данными (скриншоты → ассистенту).
5. Через Gemini собрать базу знаний и системный промпт → в `api/_lib/assistant.ts`.
6. В серверных функциях импорты — **с `.js`**.
7. Merge в `main` → импорт проекта в Vercel.
8. Добавить переменные (`TELEGRAM_BOT_TOKEN`, `GROQ_API_KEY`) → **Redeploy**.
9. BotFather → **Mini Apps** → Main App + Menu Button (корневой URL со слешем).
10. Открыть ссылку `setWebhook` для `/api/bot`.
11. Проверить мини-апп и бота. Если тихо — смотреть **Vercel → Logs**.
